# 💧 水球數學大挑戰

一個給小朋友練習算術的網頁小遊戲，靈感來自 [edugames.uz 的數學拔河遊戲](https://edugames.uz/game/arqon/)。

小朋友頭上頂著一顆水球，水會隨時間越漲越多（而且越久漲越快）；
**每答對一題，水就會減少一截**，答錯則會多加一點水。
水球漲到極限就會「啪」地爆開，遊戲結束。目標是答對越多題、撐得越久越好！

## 遊戲特色

- **單人挑戰**：一個人玩，挑戰自己的最佳紀錄（每個難度分開記錄）。
- **雙人對戰**：兩位玩家**同時上場**，左右各一個小朋友、各自的水球與題目，**誰的水球先破誰就輸**！
- **玩家名字**：單人、雙人都可以輸入名字，遊戲中顯示在小朋友的腳下（玩家 1 藍色、玩家 2 紅色），並自動記住上次輸入。
- **運算可複選**：加法、減法、乘法、除法（減法不會出現負數、除法保證整除）。
- **四種難度**：簡單／中等／困難（影響數字範圍與漲水速度），外加特殊的 **🤝 湊十好朋友**——專門練習湊成 10 的數對，可選**加法**（`A＋?=10`）、**減法**（`10−A=?`）或**混合**題型（選此難度時會自動隱藏運算選擇、改顯示湊十題型選擇）。
- **不會連續出同一題**：每次出題會避開上一題（每位玩家各自獨立判斷）。
- **選單／遊戲不同配樂**：主選單播輕柔無鼓的慢曲，進入遊戲換成輕快帶鼓的曲子，結束回選單再換回慢曲。
- **背景配樂**：WebAudio 即時合成的輕快小曲（主旋律＋貝斯＋和弦＋鼓組，8 小節循環），右上角 🎵 按鈕可隨時開關（會記住設定）。
- **緊張感升級**：水位超過 70% 進入「危險狀態」——畫面浮現脈動的紅色暈影、背景音樂逐漸加速（116→184 BPM）、小朋友開始發抖、冒冷汗、表情變緊張。
- **可愛的主角**：手繪 SVG 小朋友，大眼睛、呆毛、紅潤臉頰；平常輕鬆擺動，危險時的反應讓小朋友更有生命力。
- **音效與動畫**：答對／答錯提示音、水球爆破水花、波浪水位、快爆時水球發抖變紅。
- **輸入方式**：螢幕數字鍵盤（觸控／滑鼠），或實體鍵盤（按鍵配置見下表）。
- 支援手機直向版面（單人模式自動改為上下排列）。
- 行動裝置友善：iOS 靜音模式下也能出聲（無聲 `<audio>` 解鎖），安全區避開瀏海／home 指示條。

## 鍵盤按鍵配置

### 單人模式

| 動作 | 按鍵 |
|---|---|
| 輸入數字 | 任何數字鍵（上排或九宮格皆可） |
| 送出答案 | `Enter` |
| 刪除一個字 | `Backspace` |

### 雙人模式（一台鍵盤分兩邊）

| 動作 | 左邊玩家（玩家 1） | 右邊玩家（玩家 2） |
|---|---|---|
| 輸入數字 | 鍵盤**上排**數字 `1`~`0` | 鍵盤**右側九宮格**數字 |
| 送出答案 | `空白鍵` | `Enter`（含九宮格 Enter） |
| 刪除一個字 | `Q` | 九宮格 `.` 或 `Backspace` |

> 左手玩家集中在鍵盤左半邊、右手玩家集中在九宮格區，兩人可以同時打字互不干擾。
> 筆電沒有九宮格的話，右邊玩家可以直接點畫面上的數字鍵盤。
> 遊戲畫面下方也有各自的按鍵提示。

## 如何執行

整個遊戲就是**一個 HTML 檔**，不需要安裝任何東西：

- 直接用瀏覽器開啟 `index.html` 即可遊玩；或
- 用任何靜態伺服器，例如：

  ```
  python -m http.server 8765
  ```

  然後開啟 <http://localhost:8765>。

> 最佳紀錄、名字、配樂開關都存在瀏覽器的 `localStorage`，換瀏覽器或清除網站資料就會重置。

## 設定調整

所有可調整的數值都集中在 `index.html` 的 `<script>` 開頭，搜尋「**遊戲平衡設定**」即可找到。

### 難度參數：`DIFF`

```js
const DIFF = {
  //        數字範圍   乘除最大數  初始漲速%/s  加速度%/s²  答對扣水%
  easy:   { range: 10,  mulMax: 5,  fill0: 1.6, fillRamp: 0.030, drain: 22 },
  medium: { range: 20,  mulMax: 9,  fill0: 2.2, fillRamp: 0.045, drain: 20 },
  hard:   { range: 100, mulMax: 12, fill0: 3.0, fillRamp: 0.060, drain: 18 },
};
```

| 參數 | 意義 | 調大會怎樣 |
|---|---|---|
| `range` | 加減法的數字範圍（如 10 = 答案不超過 10） | 題目變難 |
| `mulMax` | 乘除法的最大乘數（如 5 = 最多 5×5） | 乘除變難 |
| `fill0` | 開局時水球每秒漲多少 %（水量 0~100%） | 漲水變快、更緊張 |
| `fillRamp` | 每過一秒，漲速額外增加多少（加速度） | 遊戲後期更快結束 |
| `drain` | 每答對一題，水減少多少 % | 答對的獎勵更大、更好撐 |

> `friends`（湊十好朋友）難度只有 `special: "friends"` 與 `fill0`/`fillRamp`/`drain`（水球機制），不需要 `range`/`mulMax`。題型由畫面上的「湊十題型」選擇（加法／減法／混合，存在 `friendsMode` 變數）。
>
> 題目產生在 `makeQuestion()`（純產生）與 `genQuestion()`（負責避免連續重複：重抽到和該玩家上一題 `lastQ` 不同為止，最多 20 次）。題目以「前段＋空格＋後段」呈現，所以空格能出現在中間（`A ＋ ＿ ＝ 10`）。

### 其他全域參數

| 參數 | 預設 | 意義 |
|---|---|---|
| `WRONG_PENALTY` | `6` | 答錯時水增加多少 %（設 0 = 答錯不懲罰） |
| `START_LEVEL` | `25` | 開局水量 %（越高開局越危險） |
| `DANGER_START` | `70` | 水位超過此值 %（同時觸發紅暈影、音樂加速、小朋友發抖冒汗）。調低 = 更早進入緊張狀態 |

### 配樂參數：`TRACKS` 曲目與 `music` 物件

配樂全部由 WebAudio 即時合成，無音樂檔案。共有兩首曲子放在 `TRACKS` 物件裡——`menu`（選單，輕柔無鼓、96 BPM）與 `game`（遊戲中，輕快帶鼓、116 BPM），每首是四軌的 8 小節循環：

| 每首曲目欄位 | 意義 |
|---|---|
| `tempo` | 該曲基本曲速 BPM |
| `drums` | 是否打鼓（`menu` 設 `false` 比較柔和） |
| `melody` | 主旋律：64 格八分音符陣列，填音名（如 `"C5"`）或 `0`（休止） |
| `bass` | 貝斯線：格式同上 |
| `chords` | 每小節一組三和弦，在每小節第 3、7 個八分音符輕撥 |

| `music` 參數 | 預設 | 意義 |
|---|---|---|
| `music.MAX_TEMPO` | `184` | 水位 100% 時的最快曲速 BPM（只在遊戲曲生效） |
| `music.DANGER_LV` | `DANGER_START`（70） | 水位超過此值才開始加速；曲速從該曲 `tempo` 線性升到 `MAX_TEMPO` |
| `music.VOL` | `0.5` | 配樂總音量（0~1，不影響答題音效） |

切換曲目用 `music.setTrack("menu" | "game")`：開始遊戲時切 `game`、回主選單／結束時切回 `menu`。想換曲子改對應曲目的 `melody`/`bass`/`chords` 即可，音名支援 `C0`~`B8`（含 `#`），由 `nf()` 換算頻率。

> 配樂有三層防重疊保護：
> 1. 分頁切到背景再切回來時，排程器直接跳到目前進度續播，不會把累積的音符疊在一起。
> 2. 同時開了多個遊戲分頁（或視窗）時，透過 `BroadcastChannel` 協調，**同一時間只有一個頁面出聲**（最後互動的頁面接手播放）。
> 3. 按 🔇 靜音會同步關掉**所有**分頁的配樂，且立即生效（連已排入佇列的音符也一起靜音）。

### 外觀調整

- 配色、字體、按鈕樣式都在 `<style>` 區塊內，純 CSS。
- 小朋友與水球是手繪 SVG，由 `stageSVG()` 函式產生（每位玩家一份）；想換髮色、衣服、臉部表情，直接改裡面對應的 `fill` 色碼或路徑即可。
- 水球縮放範圍在 `drawBalloon()` 內：水量 0% 時 0.55 倍、100% 時 1.18 倍。

### 危險狀態（緊張感）調整

水位超過 `DANGER_START`（70%）後，下列效果會隨「危險程度」（0→1）漸強，相關程式都在 `drawBalloon()` 與主迴圈 `loop()` 內：

| 想調整的效果 | 位置 | 怎麼改 |
|---|---|---|
| 紅色暈影的顏色／濃度 | `<style>` 的 `#dangerVignette` | 改 `radial-gradient` 的紅色與透明度 |
| 暈影脈動速度 | `loop()` 內 `pulse` | `Math.sin(t / 110)` 的 `110` 越小跳越快 |
| 音樂加速範圍 | `music.MAX_TEMPO` / `DANGER_LV` | 見上方配樂參數表 |
| 小朋友發抖幅度 | `drawBalloon()` 的 `kb` 區塊 | 抖動 `tx`／`ty` 乘上的數字越大抖越兇 |
| 冷汗滴落速度 | `drawBalloon()` 的 `dripDrop()` | `period` 越小滴越快 |
| 平常的輕鬆擺動 | `drawBalloon()` 的 `sway` | `Math.sin(t / 650)` 的幅度與週期 |

## 遊戲規則細節

- 答案輸入最多 4 位數；按「確定」或 `Enter` 送出。
- 答錯時輸入會清空，可再試同一題（但會被加水）。
- 水量下限 3%：連續答對也不會見底，遊戲一定有緊張感。
- 水量超過 70%（`DANGER_START`）進入「危險狀態」：紅色暈影、音樂加速、水球發抖變紅、小朋友發抖冒汗、表情變緊張。
- 雙人對戰：兩顆水球同速漲水（同一套難度參數），先破的一方輸；若同一瞬間一起破則平手。
- 雙人對戰不會寫入單人最佳紀錄。
